## ⚙️ Implementation Guidelines
All workspaces in this category follow the **Interpretable Context Methodology (ICM)**. This means the "Agent Architecture" is defined entirely by the folder structure. 

### How to use a Coder Workspace:
1. **Clone/Copy** the desired workspace folder (e.g., `/basher`) to your local environment.
2. **Initialize** your local LLM within that folder.
3. **Follow the Hierarchy**:
    - **Layer 0 (`AGENT.md`)**: Define your identity and orientation.
    - **Layer 1 (`CONTEXT.md`)**: Understand the global guidance.
    - **Layer 2 (`stages/`)**: Execute the sequential multi-step workflow [1].
    - **Layer 3 (`references/`)**: Adhere to the technical standards and "factory" rules.

## 🚀 Common Workflows
Workspaces in this category are designed for sequential pipelines where a human reviews the output at each boundary. Typical flows include:
`Requirement Analysis` -> `Logic Mapping` -> `Code Generation` -> `Validation`.
